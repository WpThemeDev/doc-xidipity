Palette Classes

 

media)property:value!event

media

Within the web environment content needs to be flexible and format itself appropriately to the device of the user. This is referred to a responsiveness.  Xidipity is fully responsive and uses a design language which supports mobile first. In addition, Xidipity supports all other screens dimensions under the pseudonym "sm" which means standard media.

 

property

This refers to the property of the HTML element you want to style. In the example below "text-align" and "color" are properties of the HTML paragraph element "<p>".

 

p {
  text-align: center;
  color: red;
}

 

value

The property's value is the css style to be applied. In the example above, the property value "center" instructs the browser to "text-align" center. Classes are often composed of multiple properties.

 

events

Xidipity recognizes the "hover" and "focus" events. Styling can be targeted to an event such as changing the background color when the cursor moves over the element.

palette rules

  • a property may have a media prefix
  • a property must have a value declaration suffix
  • a property value may have an event suffix

 

Examples

Class Function
bg:bas-300 Set the background color of all devices to a grey hue, shade 300
fg:pri Set the foreground color of all devices to the primary standard hue
sm)fg:bas-800 Set the foreground color of standard media devices to the base hue, shade 800
bg:sec!hover Set the background color of all media devices to the secondary standard hue during the hover event
bdr:green Set the border color of all media devices to the google green hue
