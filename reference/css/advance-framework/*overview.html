Advance Classes

 

environment[media)property:value!event]

 

The purpose of Advance CSS classes is to provide a method whereby authors or web developers can extend the functionality and styling of Xidipity templates. Although Advance classes may resemble a CSS framework, they are limited in scope to enhance performance. The goal of the syntax is to make their use intuitive.

 

environment

print

Abbreviation: prt

The print environment is content formatted for printing. This environment is supported by the "sys/print-var(css)" and the "theme/print(css)" files. The default assumption is print output will be monochrome.

 

web

Abbreviation: web

The web environment is content formatted for devices with a screen. This is the default environment and only needs to be specified in limited circumstances.

 

editor

Abbreviation: mce

The editor environment is a subset of the web environment with adjustments made to enhance content creation. Like the web environment, it only needs to be specified in limited circumstances.

 

media

Within the web environment content needs to be responsive and format appropriately to the device of the user. This is referred to as responsiveness.  Xidipity is fully responsive and uses a design language which supports mobile first. In addition, Xidipity supports all other screens dimensions under the pseudonym "sm" which means standard media. The default assumption is the class will be applied to all unless prefixed with "sm)".

 

property

This refers to the property of the HTML element you want to style. In the example below "text-align" and "color" are properties of the HTML paragraph element "<p>".

 

p {
  text-align: center;
  color: red;
}

 

value

The property's value is the css style to be applied. In the example above, the property value "center" instructs the browser to center (align) the content between the paragraph tags. Classes are often composed of multiple properties.

 

event

CSS recognizes several events. Examples include "hover", "focus", and "active". Xidipity supports hover and focus. Styling can be targeted to an event such as changing the background color when the cursor moves over the element.

 

advance rules

  • a property may have an environment prefix
  • a property may have a media prefix
  • a property may have an environment and a media prefix
  • a property must have a value declaration suffix
  • a property value may have an event suffix
